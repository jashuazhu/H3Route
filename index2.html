1. Background

This project introduces H3-Route, a priority-aware online routing heuristic for hazard-zone relief operations using autonomous robots and drones. The method targets dynamic request streams with goals of bounded decision latency, route stability, and near-optimal travel distance—all critical for vehicles constrained by fuel or battery capacity.

The core idea builds a 3-D “priority-time warp” using a Hilbert space-filling curve that fuses geography with urgency metadata for efficient, locality-preserving ordering. A frozen horizon of imminent stops maintains stability, while periodic, bounded local optimization removes detours, outperforming classic greedy and FIFO baselines in simulation studies.

2. The Idea

2.1 — 3-D “Priority-Time Warp” via Hilbert Space-Filling Curve (H)

  • Multi-Dimensional Key. Normalize geo-coordinates (x, y) and convert priority/deadlines to a scalar urgency U. Use a Hilbert SF curve to map (x, y, U) to a locality-preserving index.
  • Total Ordering. The resulting 3-D Hilbert index D defines a single urgency-aware, geometry-respecting order for requests.
  • Dynamic Sequencing. Sorting by D creates a “warp” where urgent nodes bubble forward without global re-solve.

2.2 — Efficient Online Insertion with Priority-Aware Regret (H)

  • Data Structure. Maintain route order in a balanced structure (e.g., augmented skip-list or RB-tree) keyed by the Hilbert index to find proximal neighbors in O(log n).
  • Candidate Set. Score only a small number k of nearby edges (u, v) to evaluate insertions.
  • Regret Objective. Combine (i) added distance/energy, (ii) priority-weighted lateness if ETA exceeds deadline, and (iii) a stability penalty for changes inside a frozen prefix of length F.
  • Per-Insert Complexity. O(log n + k), effectively near-constant when k is small.
Sketch of the priority-aware regret
For candidate edge (u, v) and new request r with priority weight w(pr) and deadline dr:
Regret(u,v,r) = ΔDistance(u,r,v) + w(p_r)·max{0, ETA_r − d_r} + λ·1{position(u→r→v) ≤ F}
  • ΔDistance: additional travel cost by inserting r between u and v.
  • w(pr): larger for higher emergencies; converts lateness into commensurate penalty.
  • λ: stability weight discouraging edits in the frozen prefix of length F.

2.3 — Periodic Receding-Horizon Optimization & Stability Prefix (H)

  • Stability Mechanism. Freeze the next F imminent stops to protect controllers from route thrashing during request spikes.
  • Local Cleanup. Every τ inserts, run a bounded local search (e.g., 2-opt/3-opt/Or-opt) on a small window after the prefix to prune detours.
  • Near-Static-TSP Quality. Receding the window forward yields high-quality routes without a costly global re-solve.

3. The Impact

3.1 — Saving Lives Through Timely, Efficient Response

  • Urgency First. The priority-time warp elevates urgent nodes while respecting deadlines, improving time-to-aid.
  • Energy-Aware Efficiency. Near-shortest routing conserves fuel/battery, extending mission reach and dwell time.

3.2 — Stability for Reliable Operations

  • Flight/Drive Safety. The frozen prefix reduces churn, producing predictable motion plans for embedded controllers.
  • Bounded Latency. Fast O(log n + k) insertions prevent compute bottlenecks from delaying aid.

3.3 — Scaling via Fleet Coordination

  • Multi-Vehicle Extension. Partition the Hilbert index space for assignment; manage cross-vehicle hand-offs.
  • Coverage & Throughput. Coordinated fleets allocate resources efficiently over wide areas in large-scale disasters.
